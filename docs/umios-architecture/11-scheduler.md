## FPU コンテキスト退避ポリシー

Cortex-M4F の FPU レジスタ（S0–S31, FPSCR）はタスク切替時に退避/復元が必要になる場合がある。ポリシーはコンパイル時にテンプレートパラメータで選択する（ポリシーベース設計）。

| ポリシー | 動作 | 用途 |
|---------|------|------|
| Exclusive | 常に FPU レジスタを保存/復元 | そのタスクが FPU を独占的に使用する場合 |
| LazyStack | Cortex-M Lazy Stacking を利用。実際に FPU を使用した場合のみ退避 | 複数タスクが FPU を使いうる場合（推奨） |
| Forbidden | FPU レジスタの退避を行わない。FPU を使うと他タスクの状態を破壊する | FPU を一切使わないタスク |

### 設計上の注意

- 実際には複数タスクが FPU を使いたい場面が多い（Audio Task の DSP 処理、EventRouter の float 演算等）
- 2 つ以上のタスクが FPU を使うなら LazyStack が必須。Forbidden で済むのは Idle Task 程度
- **FPU なしターゲット**（Cortex-M0 等）ではこのポリシー自体が無効になる。`#if __FPU_PRESENT` で条件コンパイル

### FPU なし/あり版のコード設計方針

- 基本は FPU なしでも動作するコードを書く
- FPU あり版は最適化パスとして用意（`#if __FPU_PRESENT` 等で分岐）
- DSP ライブラリ（umidsp）は float 版と固定小数点版を提供する方針
- ドキュメント内のコード例が float 前提の場合、FPU 前提であることを注記する
