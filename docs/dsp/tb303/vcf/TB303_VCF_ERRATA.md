# TB-303 VCF ドキュメント検証レポート

## 概要

本ドキュメントは、TB303_VCF_COMPLETE.md および TB303VCF2.md の内容を検証し、発見された誤りと修正計画をまとめたものである。

---

## 1. 重大な誤り

### 1.1 【致命的】最適化実装のトポロジーが異なる

**場所**: TB303_VCF_COMPLETE.md セクション8.2、付録A

**問題**: `DiodeLadderOptimized` および `DiodeLadder` クラスは、名前に「DiodeLadder」とあるが、**ダイオードラダーのトポロジーではなく、一般的なMoog型トランジスタラダーのTPT実装**である。

**証拠**:

| 要素 | ダイオードラダー（Karrikuh） | 問題の実装（DiodeLadderOptimized） |
|------|---------------------------|----------------------------------|
| 状態寄与係数 | `a³c`, `a²bc`, `(b²-2a²)ac`, `(b²-3a²)bc` | `G³`, `G²`, `G`, `1`（単純累乗） |
| 伝達関数分母 | `2a⁴ - 4a²b² + b⁴` | `(1+G)⁴`ベース |
| フィードバックHPF | `z[4]`, `ah`, `bh`で内蔵 | **なし**（完全に欠落） |
| 周波数特性 | 非対称（C₁=C/2による「18dB/oct」的） | 対称（24dB/oct均一） |

**Karrikuh原実装の状態寄与**:
```cpp
const double s0 = (a2 * a * z[0]           // a³c
                 + a2 * b * z[1]           // a²bc
                 + z[2] * (b2 - 2 * a2) * a // (b²-2a²)ac
                 + z[3] * (b2 - 3 * a2) * b // (b²-3a²)bc
                 ) * c;
```

**問題の実装の状態寄与**:
```cpp
const F S = omG_ * (s3_ + G_*(s2_ + G_*(s1_ + G_*s0_)));
// = (1-G) * (G³s₀ + G²s₁ + Gs₂ + s₃) ← Moogラダーの形式
```

**影響度**: 致命的
**原因**: Moog型TPTラダーの実装をダイオードラダーと誤認

---

### 1.2 【致命的】フィードバックHPFの欠落

**場所**: TB303_VCF_COMPLETE.md セクション8.2、付録A

**問題**: TB-303の「アシッドサウンド」の核心であるフィードバックHPF（92Hz）が、最適化実装に完全に欠落している。

**Karrikuh原実装**:
```cpp
// フィードバックHPF適用
const double s = bh * s0 - z[4];

// HPF状態更新
z[4] = bh * y4 + ah * y5;
```

**問題の実装**: フィードバックHPFに相当するコードが**一切存在しない**。

**影響度**: 致命的
**原因**: TPT形式への「変換」時にフィードバックHPFの移植を失念

---

### 1.3 【重大】付録Cの対応表が不正確

**場所**: TB303_VCF_COMPLETE.md 付録C

**問題**: 「Karrikuh原実装との対応」を示す表が、実際には対応していない実装同士を比較している。

**不正確な記述**:
| Karrikuh原実装 | 最適化実装 | 説明 |
|----------------|-----------|------|
| `c = 1 / (2*a4 - 4*a2*b2 + b4)` | `rcp = 1 / (1 + k * G4)` | 分母の逆数 |

**事実**: `c`はダイオードラダー特有の伝達関数分母、`rcp`はMoogラダーのフィードバック除算。数学的に**全く異なる**。

**影響度**: 重大
**原因**: 異なるアルゴリズムを「対応」と誤記

### 2.2 inv_sqrt2補正がTPT実装に反映されていない

**場所**: TB303_VCF_COMPLETE.md セクション8.2

**問題**: セクション4で`inv_sqrt2`（1/√2）補正の重要性を詳述しているにもかかわらず、最適化実装には一切反映されていない。

**TB303VCF2.mdの正しい実装**:
```cpp
const F a = std::tan(pi<F> * wc_) * kTb303Scale;  // kTb303Scale = 1/√2
```

**問題の実装**:
```cpp
const F g = std::tan(F(M_PI) * fc_ / sr_);  // inv_sqrt2なし
```

**影響度**: 重大
**原因**: TPT形式への変換時に補正係数を失念

## 3. 不正確な記述

### 3.1 セクション7のTPT変換式

**場所**: TB303_VCF_COMPLETE.md セクション7.3

**問題**: 「Karrikuhの$s_0$に対応するTPT形式」として以下を記載しているが、これは**対応していない**。

```
S = (1-G) · (G³s₀ + G²s₁ + Gs₂ + s₃)
```

**事実**: Karrikuhの状態寄与係数は`a³c`, `a²bc`, `(b²-2a²)ac`, `(b²-3a²)bc`であり、`G³`, `G²`, `G`, `1`ではない。この差異はダイオードラダー特有の非対称キャパシタ配置（C₁=C/2）に由来する。

**正しい記述**: 「TPT形式のMoogラダーの状態寄与」であり、ダイオードラダーとは異なる。

**影響度**: 中程度
**原因**: MoogラダーとダイオードラダーのTPT形式の混同

---

### 3.2 セクション8.3の性能比較表

**場所**: TB303_VCF_COMPLETE.md セクション8.3

**問題**: 「参照実装」「最適化実装」と「Karrikuh原版」を同列に比較しているが、アルゴリズムが異なるため比較は不適切。

**不適切な記述**:
| 実装 | 周波数精度 |
|------|------------|
| Karrikuh | △（ワーピングなし）|
| 参照/最適化 | ◎（TPT）|

**事実**: Karrikuhと参照/最適化は**異なるフィルタートポロジー**を実装しているため、精度比較は意味をなさない。

**影響度**: 中程度
**原因**: 異なるアルゴリズムの混同

---

### 3.3 クリッピング関数の違い

**場所**: TB303_VCF_COMPLETE.md セクション8.2

**問題**: Karrikuhは`x / (1 + |x|)`を使用、最適化実装は`fast_tanh()`を使用。これらは異なる非線形特性を持つ。

| 関数 | 飽和特性 | 用途 |
|------|----------|------|
| `x / (1 + |x|)` | ソフト、対称 | Karrikuh原実装 |
| `tanh(x)` | より急峻 | 一般的なVA |
| `clamp(x, -1, 1)` | ハード | Tekitou実装 |

**影響度**: 軽微（聴覚的差異は小さい）
**原因**: 意図的な選択の可能性もあるが、説明なし

ダイオードラダーフィルターの実装は、以下を満たす必要がある：

- [ ] 状態寄与係数が`a³c`, `a²bc`, `(b²-2a²)ac`, `(b²-3a²)bc`形式
- [ ] 伝達関数分母が`2a⁴ - 4a²b² + b⁴`
- [ ] フィードバックHPF（92Hz / 578.1 rad/s）が存在
- [ ] inv_sqrt2（1/√2）補正が適用されている（TB-303特有）
- [ ] 出力補償（`A = 1 + 0.5*k`またはCompensation HPF）が存在
- [ ] 2xオーバーサンプリング対応（またはtan()によるワーピング補正）
