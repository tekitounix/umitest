# ポテンショメータ入力処理

| 項目 | 値 |
|------|-----|
| ターゲット | 電子楽器 (SAR型ADC搭載MCU / 高インピーダンス回路) |
| 目標 | パルスノイズ除去、クロストーク回避、パラメータの高分解能化 |
| 手法 | オーバーサンプリング + メディアンフィルタ + 2回連続読み (Double Sampling) |
| 理由 | 高インピーダンス回路における「パルスノイズ」と「チャンネル間クロストーク」を物理・ソフト両面で除去するため |

---

> 共通の設計方針（ISR/処理ループの分離、周期設計など）は [README.md](README.md) を参照。

## 概要

SAR型ADCを持つMCU全般において、インピーダンスが高いポテンショメータをオペアンプバッファなしで接続し、シーケンシャル読み出しを行う場合、内部コンデンサ（$C_{sh}$）に前のチャンネルの電荷が残ったまま次の測定が行われることで、深刻なクロストーク（ゴースト）が発生します。以下ではSTM32を例に説明しますが、原理と対策はSAR型ADCを持つ他のMCU（RP2040、nRF52、ESP32等）にも共通して適用できます。

ハードウェアを変更せずにこの問題を解決するための対策と、推奨される構成を以下にまとめます。

## 一般的なサンプリング方法

Timer割り込み、またはTimer駆動のDMAで周期的にサンプリングを行う。最もシンプルなのはこの値をそのまま用いる方法ですが、ノイズやクロストークの影響を受けやすいため、以下の対策を講じることを推奨します。

- ADCを10kHz程度で高速サンプリングしCircular DMAで複数サンプルをバッファリング。ポテンショメータの操作帯域は通常10〜20Hz程度だが、10kHzまでオーバーサンプリングすることでフィルタリングによるノイズ低減効果（$\sqrt{N}$倍のSNR改善）と、Double Sampling等のクロストーク対策に必要なスループットを確保できる
- 処理ループで固定周期(**1kHz**)でDMAバッファを窓単位で処理する。窓に対してメディアン、EMA等のフィルタリングを適用し、状態変化イベントを生成する
  - 1kHzの根拠: ポテンショメータの操作帯域は10〜20Hz程度であり、ナイキスト条件（40Hz以上）を大幅に上回る。1kHz処理なら操作に対して十分なイベント分解能があり、かつ人間が知覚できない1msの更新周期で滑らかな追従が得られる
  - ボタンやMIDI UARTの処理ループと同じ1kHz周期で統一でき、共通の周期処理として駆動できる

## 課題と対策方針

SAR型ADCは入力インピーダンスの影響を受けやすく、以下の問題が発生しやすい。※内部パスに問題を抱えたエラッタ扱いのMCUも存在する。

- **パルスノイズ:** デジタル回路からの飛び込みや接触不良によるスパイク
- **クロストーク (Ghosting):** シーケンシャル読み出し時に前のチャンネルの電圧が残留する

**対策:** サンプリング時間を物理的に最大化し、ソフトウェアで外れ値を除去する。

## ハードウェア / ADC設定

### サンプリング時間 (Sampling Time)

- **設定:** 使用するMCUのADCが許容する最大サイクルに設定する
- **推奨値:** 例としてSTM32F4では480 cycles、STM32H7では640.5 cyclesなど（シリーズごとに異なる）
- **理由:** 内部コンデンサ($C_{sh}$)への充填時間を確保し、電圧降下と他チャンネルの影響を排除するため
- **一般原則:** SAR型ADCを持つMCU全般で、ポテンショメータのような高インピーダンスソースに対してはサンプリング時間を最大化する方針は共通

### ADCクロック (ADCCLK)

- **設定:** 低速化 (Prescaler)
- **推奨:** ADCの変換速度を落とし、物理的なサンプリング時間（$\mu s$）を長くする
- **理由:** サイクル数が同じでも、クロックが遅ければ充填時間は長くなるため

MCUのクロックツリー設定で、ADCプリスケーラを大きく設定し、ADC自体の駆動クロックを下げる。

**計算例（STM32の場合）:**
- **14MHz** で 480 cycles ≈ 34 $\mu$s
- **4MHz** で 480 cycles ≈ 120 $\mu$s

**効果:** サンプリング時間が物理的に延びるため、高インピーダンスでもコンデンサへの充填が間に合うようになる。

### シーケンサ設定 (Double Sampling)

#### 通常のシーケンス

`[CH1] -> [CH2] -> [CH3] ...`

CH1の残留電荷（例: 3.3V）がCH2（例: 0V）測定時に放電しきれず、値が浮いてしまう。

#### 対策1: 多重サンプリング・ドロップ法

`[CH1] -> [CH1] -> [CH2] -> [CH2] -> [CH3] -> [CH3] ...`

1. **1回目の CH2:** 「捨て石」。CH1の残留電荷を$C_{sh}$から追い出し、CH2の電圧に近づけるためのプリチャージとして利用
2. **2回目の CH2:** $C_{sh}$が既にCH2の電圧に近い状態でサンプリングするため、正確な値が取れる

#### 対策2: ガードバンド法

`[CH1] -> [VREFINT] -> [CH2] -> [VREFINT] -> [CH3] ...`

チャンネル間に、インピーダンスが極めて低い（電流を吸い込める）内部チャンネルを挟む方法。

- **使用チャンネル:** `VREFINT` や `GND (VSSA)`
- **効果:** インピーダンスが低いチャンネルを読むことで、前のチャンネルで溜まった電荷を急速に引き抜いてリセット（ウォッシュアウト）する
- **適性:** 対策1（2回読み）でもクロストークが取りきれないほどインピーダンスが高い（50kΩ〜100kΩ以上）場合に有効

#### 各対策のメリット・デメリット

- **メリット:** ハードウェア変更不要で高い効果が得られる
- **デメリット:** スループット（実効サンプリングレート）が半分になる（ポテンショメータ用途では精度優先のため通常は問題なし）

## ソフトウェア後処理 (DSP / Task)

### データ取得 (DMA)

- **モード:** Circular Mode
- **バッファ:** リングバッファを使用し、CPUは常に最新のブロックを参照する

### フィルタリングパイプライン

処理ループの周期（例: 1kHz）ごとに以下の順で処理を行う。

1. **メディアンフィルタ (Median Filter) [必須]**
   - DMAバッファ内の同一チャンネルの最新N個（推奨: 5〜9サンプル）を取得し、ソートして中央値を採用する。例えばADC 10kHz / 処理ループ 1kHzなら1周期あたり約10サンプルが蓄積されるので、その中からN個を使う
   - パルス状のスパイクノイズ（外れ値）を完全に無視できる。平均値フィルタではノイズに引きずられるため、メディアンが最適

2. **IIR / EMAフィルタ (Smoothing)**
   - メディアン処理後の値に対して、指数移動平均をかける
   - 式: $y[n] = \alpha \cdot x[n] + (1 - \alpha) \cdot y[n-1]$
   - ADCの量子化ノイズを抑え、操作感を滑らかにする。電子楽器のパラメータとしては必須
   - **αの目安:** 0.05〜0.3 程度。小さいほど滑らかだが応答が遅くなる。処理ループ周期1kHzの場合、α=0.1で時定数約10ms（ノブ操作に対して自然な追従感）、α=0.02で時定数約50ms（非常に滑らか）。用途に応じて調整する

3. **ヒステリシス（デッドゾーン）判定**
   - EMAフィルタ後の値と前回出力値の差が閾値（例: 12bit ADCなら±2〜4 LSB）を超えた場合のみイベントを発行する
   - これがないとADCの量子化ノイズやEMAの端数で値が±1揺れ続け、不要なイベントが大量発生する
   - 楽器用途ではパラメータの「静止」を保証するために必須

> **補足:** もし回路設計が可能なら、ポテンショメータの直後にオペアンプ（ボルテージフォロワ）を入れるのが物理的解決策としては最強。しかし、既存基板の対策としては上記ソフトウェア構成が最適解となる。
