# ポテンショメータ入力処理 実装戦略
**ターゲット:** 電子楽器 (STM32 / 高インピーダンス回路)
**目標:** パルスノイズ除去、クロストーク回避、パラメータの高分解能化
**手法:** オーバーサンプリング + メディアンフィルタ + 2回連続読み (Double Sampling)
**理由:** 高インピーダンス回路における「パルスノイズ」と「チャンネル間クロストーク」を物理・ソフト両面で除去するため。

---

## 概要
STM32のADCにおいて、インピーダンスが高いポテンショメータをオペアンプバッファなしで接続し、シーケンシャル読み出しを行う場合、内部コンデンサ（$C_{sh}$）に前のチャンネルの電荷が残ったまま次の測定が行われることで、深刻なクロストーク（ゴースト）が発生します。

ハードウェアを変更せずにこの問題を解決するための3つの対策と、推奨される構成を以下にまとめます。

---

## 1. 一般的な方法
### サンプリング
Timer割り込み、またはTimer駆動のDMAで周期的にサンプリングを行う。最もシンプルなのはこの値をそのまま用いる方法ですが、ノイズやクロストークの影響を受けやすいため、以下の対策を講じることを推奨します。

- ADCを10kHzなど高速でサンプリングしCircular DMAで複数サンプルをバッファリング
- サーバータスクで固定周期(1kHz程度)でDMAバッファを窓単位で処理する。窓に対して平均、メディアン、移動平均、IIRなどのフィルタリングを適用し、状態変化イベントとともに共有メモリに書き込み

## 2. 課題と対策方針
SAR型ADCは入力インピーダンスの影響を受けやすく、以下の問題が発生しやすい。※内部パスに問題を抱えたエラッタ扱いのMCUも存在する。

* **パルスノイズ:** デジタル回路からの飛び込みや接触不良によるスパイク。
* **クロストーク (Ghosting):** シーケンシャル読み出し時に前のチャンネルの電圧が残留する。

**対策:** サンプリング時間を物理的に最大化し、ソフトウェアで外れ値を除去する。

## 2. ハードウェア / ADC設定

### A. サンプリング時間 (Sampling Time)
* **設定:** **最大サイクル (Max Cycles)**
    * *推奨値:* 480 cycles, 640.5 cycles など（シリーズの最大値）
    * *理由:* 内部コンデンサ($C_{sh}$)への充填時間を確保し、電圧降下と他チャンネルの影響を排除するため。

### B. ADCクロック (ADCCLK)
* **設定:** **低速化 (Prescaler)**
    * *推奨:* ADCの変換速度を落とし、物理的なサンプリング時間（$\mu s$）を長くする。
    * *理由:* サイクル数が同じでも、クロックが遅ければ充填時間は長くなるため。
#### 設定方法
STM32のクロックツリー設定で、ADCプリスケーラを大きく設定し、ADC自体の駆動クロックを下げます。

#### 計算例
* **14MHz** で 480 cycles $\approx$ 34 $\mu$s
* **4MHz** で 480 cycles $\approx$ 120 $\mu$s

**効果**: サンプリング時間が物理的に延びるため、高インピーダンスでもコンデンサへの充填が間に合うようになります。

### C. シーケンサ設定 (Double Sampling)
#### 通常のシーケンス
`[CH1] -> [CH2] -> [CH3] ...`
- CH1の残留電荷（例: 3.3V）がCH2（例: 0V）測定時に放電しきれず、値が浮いてしまう。

#### 対策1：多重サンプリング・ドロップ法
`[CH1] -> [CH1] -> [CH2] -> [CH2] -> [CH3] -> [CH3] ...`
1.  **1回目の CH2**: 「捨て石」です。CH1の残留電荷を$C_{sh}$から追い出し、CH2の電圧に近づけるためのプリチャージとして利用します。
2.  **2回目の CH2**: $C_{sh}$が既にCH2の電圧に近い状態でサンプリングするため、正確な値が取れます。

#### 対策2：ガードバンド法
`[CH1] -> [VREFINT] -> [CH2] -> [VREFINT] -> [CH3] ...`
チャンネル間に、インピーダンスが極めて低い（電流を吸い込める）内部チャンネルを挟む方法です。
* **使用チャンネル**: `VREFINT` や `GND (VSSA)`
* **効果**: インピーダンスが低いチャンネルを読むことで、前のチャンネルで溜まった電荷を急速に引き抜いてリセット（ウォッシュアウト）します。
* **適性**: 対策1（2回読み）でもクロストークが取りきれないほどインピーダンスが高い（50kΩ〜100kΩ以上）場合に有効です。

#### 各対策のメリット・デメリット
* **メリット**: ハードウェア変更不要で高い効果が得られる。
* **デメリット**: スループット（実効サンプリングレート）が半分になる。（※ポテンショメータ用途では精度優先のため通常は問題なし）

## 3. ソフトウェア後処理 (DSP / Task)

### A. データ取得 (DMA)
* **モード:** Circular Mode
* **バッファ:** リングバッファを使用し、CPUは常に最新のブロックを参照する。

### B. フィルタリングパイプライン
タスク周期（例: 1kHz）ごとに以下の順で処理を行う。

1.  **メディアンフィルタ (Median Filter) [必須]**
    * *処理:* バッファ内の最新 `N個` (推奨: 5～9個) を取得し、ソートして**中央値**を採用する。
    * *効果:* パルス状のスパイクノイズ（外れ値）を完全に無視できる。平均値フィルタではノイズに引きずられるため、メディアンが最適。

2.  **IIR / EMAフィルタ (Smoothing)**
    * *処理:* メディアン処理後の値に対して、指数移動平均をかける。
    * *式:* $y[n] = \alpha \cdot x[n] + (1 - \alpha) \cdot y[n-1]$
    * *効果:* ADCの量子化ノイズを抑え、操作感を滑らかにする。電子楽器のパラメータとしては必須。

> **補足**:
> もし回路設計が可能なら、ポテンショメータの直後にオペアンプ（ボルテージフォロワ）を入れるのが物理的解決策としては最強です。しかし、既存基板の対策としては上記ソフトウェア構成が最適解となります。
